# 三问登峰作答页 · 公网部署说明

> 目标:7/30 罗浮山活动当天,23 人用手机 4G 扫码 → 访问页面 → 匿名提交三问 → 回响墙实时看所有人答案。
> 关键约束:山上不在同一 WiFi,必须公网访问;答案要 23 人共享,必须有后端存储。

## 架构与文件

```
作答页/
├── server.js        # 后端(零第三方除 pg 外无依赖)
├── store.js         # 存储抽象:无 DATABASE_URL→answers.json;有 DATABASE_URL→PostgreSQL
├── package.json     # 依赖 pg,启动脚本 npm start
├── render.yaml      # render 一键 Blueprint 配置(可选)
├── public/
│   └── index.html   # 前端 SPA:首页/?v=q&q=1/2/3 答题、?v=wall 回响墙
└── answers.json     # 本地开发用数据文件(生产用 PG,不写此文件)
```

## 为什么不能直接文件存(render 限制)

render 免费层文件系统是**临时的**(ephemeral):服务 15 分钟无请求会休眠,再次唤醒=重启容器,运行时写入的 `answers.json` 会被清空。重新部署也会清空。所以生产必须接持久数据库。本地开发无此问题,用 `answers.json` 即可。

## 路径 A:render + Supabase(推荐 · 免费 · 公网 HTTPS · 约 20 分钟)

全程两个免费账号:render(跑 Node)+ Supabase(存数据,免费 PostgreSQL)。都不收费、不绑卡。

### 1. Supabase 建库(5 分钟)

1. 注册 https://supabase.com(可用 GitHub 登录),New Project,起名 `sanwen`,选 region(建议 Northeast Asia - Tokyo / Singapore,国内访问较近)。
2. 设数据库密码(记牢)。
3. 项目创建后:**Project Settings → Database → Connection string → Transaction pooler(端口 6543)**,复制 URI,形如:
   ```
   postgresql://postgres.xxxxx:密码@aws-0-ap-northeast-1.pooler.supabase.com:6543/postgres
   ```
   把其中 `[你的密码]` 填回真实密码。**这个串就是 `DATABASE_URL`。**
4. 表会自动建(`store.js` 启动时 `CREATE TABLE IF NOT EXISTS answers`),无需手动建表。

> Supabase 免费层 500MB 存储、连接数充足,活动 23 人绰绰有余。无需绑卡。

### 2. 推代码到 GitHub(3 分钟)

1. 把整个 `作答页/` 目录推到一个 GitHub 仓库(私有即可),例如 `sanwen-dengfeng`。
2. 仓库根目录就是 `作答页/` 的内容(能直接看到 `server.js`、`package.json`)。

### 3. render 创建服务(5 分钟)

1. 注册 https://render.com(用 GitHub 登录,免单独注册)。
2. New → **Web Service** → 选刚推的 GitHub 仓库。
3. 配置:
   - **Name**:`sanwen-dengfeng`
   - **Runtime**:Node
   - **Build Command**:`npm install`
   - **Start Command**:`npm start`
   - **Plan**:**Free**
4. 环境变量(Environment):
   - `DATABASE_URL` = 第 1 步复制的 Supabase 连接串
   - `ADMIN_KEY` = 自定义一串(如 `sanwen2026reset`,用于事后清空数据,别外泄)
5. Create Web Service。render 自动 `npm install` + 启动,等 1-2 分钟。
6. 拿到地址:`https://sanwen-dengfeng.onrender.com`

### 4. 验证

浏览器访问:
```
https://sanwen-dengfeng.onrender.com/api/health
```
返回 `{"ok":true,"service":"sanwen-dengfeng","time":...,"db":true}` 即正常(`db:true` 证明已连上 PG)。

再访问 `https://xxx.onrender.com/` 看到首页三问入口、`/?v=q&q=1` 答题页、`/?v=wall` 回响墙,即部署成功。

## 路径 B:公司公网服务器(备选 · 最稳 · 需有服务器)

若公司有一台带公网 IP/域名的机器,直接跑,不用 render:

```bash
# 装 Node >=16,把作答页目录放上去
cd /opt/sanwen && npm install
# 配 PostgreSQL(本机或云),导出连接串
export DATABASE_URL="postgresql://user:pass@host:5432/db"
export ADMIN_KEY="sanwen2026reset"
node server.js   # 或 pm2 start server.js --name sanwen && pm2 save && pm2 startup
```

nginx 反代到 80/443(配 HTTPS,微信扫码要求 HTTPS 才不拦截):

```nginx
server {
    listen 443 ssl;
    server_name sanwen.你公司域名.com;
    # ssl_certificate / ssl_certificate_key ...
    location / {
        proxy_pass http://127.0.0.1:8788;
        proxy_set_header X-Forwarded-For $remote_addr;
        proxy_set_header Host $host;
    }
}
```

验证同上 `/api/health`。

## 活动当天保活(关键!render 免费层防休眠)

render 免费层 **15 分钟无请求会休眠**,下次访问冷启动约 30-60 秒,体验差。务必保活:

1. 注册 https://uptimerobot.com(免费),Add New Monitor。
2. Monitor Type:**HTTP(s)**,URL 填 `https://sanwen-dengfeng.onrender.com/api/health`,间隔 **5 minutes**。
3. **活动当天上午开启**,确保全天不休眠;活动结束后可关掉省配额。

> 备选:现场任何一台手机开着回响墙页面(每 8 秒自刷)也能保活,双保险。

## 接口一览

| 接口 | 方法 | 说明 |
|---|---|---|
| `/` | GET | 首页(三问入口) |
| `/?v=q&q=1/2/3` | GET | 第 1/2/3 问作答页 |
| `/?v=wall` | GET | 回响墙(8 秒自刷) |
| `/api/answer` | POST | 提交答案 `{q, text}` |
| `/api/answers` | GET | 全部答案(已匿名,剥离 IP) |
| `/api/stats` | GET | 各问参与人数 `{target, q1, q2, q3}` |
| `/api/health` | GET | 健康检查(含 `db` 标志) |
| `/api/reset` | POST | 清空数据(需 header `X-Admin-Key: <ADMIN_KEY>`) |

清空数据(活动后或彩排后重置):
```bash
curl -X POST https://sanwen-dengfeng.onrender.com/api/reset -H "X-Admin-Key: sanwen2026reset"
```

## 数据备份

- 生产数据在 Supabase 的 `answers` 表。Supabase 控制台 → Table Editor → answers 可直接看/导出 CSV。
- 也可 `curl https://xxx/api/answers > backup.json` 导出全量。

## 匿名性

前端与 `/api/answers` 公开接口均不返回 IP。IP 仅存于数据库 `ip` 字段,用于同人同问防重复,不对外。

## 常见问题

- **首次访问很慢(30s+):**render 免费层冷启动,用 UptimeRobot 保活后无此问题。
- **提交返回 429:**同 IP 1 分钟超 10 次被限频,正常活动不会触发。
- **回响墙不刷新:**8 秒自动重试,网络恢复自动恢复,无需手动操作。
- **二维码扫不开:**微信对纯 HTTP 会拦截,务必用 HTTPS(render 默认已是 HTTPS)。
